**Issues** : #130 (FileEvent enrichi) · #131 (FileInfo.Version)

## 1. Contexte

Ces extensions enrichissent les structures partagées `FileEvent` et `FileInfo` de `plugins/plugin.go` pour permettre une invalidation de cache plus précise (consommé par le cache de chunks v2.1, issue #124).

**Contrainte** : aucun BREAKING CHANGE — tous les champs sont optionnels (zero-value valide).

---

## 2. FileInfo — champ `Version` (#131)

### Modification de la struct

```go
type FileInfo struct {
    Name          string    `json:"name"`
    Path          string    `json:"path"`
    Size          int64     `json:"size"`
    IsDir         bool      `json:"isDir"`
    ModTime       time.Time `json:"modTime"`
    ETag          string    `json:"etag"`
    IsPlaceholder bool      `json:"isPlaceholder"`
    IsCached      bool      `json:"isCached"`
    // Version is an opaque backend-provided version token used for cache
    // invalidation. Semantics are backend-specific:
    //   - WebDAV  : same value as ETag (HTTP entity tag)
    //   - MooseFS : Unix ctime as decimal string — changes on any attribute
    //               or content modification (more precise than mtime alone)
    //   - local   : "" — no native versioning; callers fall back to ModTime
    // An empty Version is always valid; consumers must fall back to ModTime.
    Version string `json:"version,omitempty"`
}
```

### Sémantique par backend

| Backend | `Version` | Valeur exemple | Fallback |
|---------|-----------|----------------|---------|
| WebDAV  | ETag (HTTP entity tag) | `"33a64df5"` | ModTime |
| MooseFS | CTime Unix (secondes) décimal | `"1716198432"` | ModTime |
| local   | `""` (non disponible) | — | ModTime |
| Plugins tiers | opaque ou `""` | — | ModTime |

**Règles** :
- Un consumer ne doit jamais interpréter la valeur interne de `Version`
- La comparaison `old.Version != new.Version` suffit pour détecter un changement
- Si `Version == ""` → fallback obligatoire sur `ModTime`

## 3. FileEvent — champs enrichis (#130)

### Modification de la struct

```go
type FileEvent struct {
    Type      FileEventType `json:"type"`
    Path      string        `json:"path"`
    OldPath   string        `json:"oldPath,omitempty"`
    Timestamp time.Time     `json:"timestamp"`
    Source    string        `json:"source"` // "local" | "remote"

    // ModTime is the file's new modification time as reported by the backend
    // at the moment the change was detected. Zero value when not available
    // (e.g. for FileEventDeleted or FileEventRenamed).
    // Allows consumers to skip a Stat() call after receiving a Watch event.
    ModTime time.Time `json:"modTime,omitempty"`

    // PreviousModTime is the file's modification time from the previous
    // snapshot, allowing detection of a modification without an additional
    // Stat() call. Zero when the file is newly created or the previous mtime
    // is unknown.
    PreviousModTime time.Time `json:"previousModTime,omitempty"`

    // MetadataOnly is true when only the file's metadata changed (ETag/mtime)
    // without a content change (size is unchanged). When true, the cache layer
    // may invalidate metadata-only entries without evicting content chunks.
    MetadataOnly bool `json:"metadataOnly,omitempty"`
}
```

### Nouveau type d'événement

```go
const (
    // FileEventMetadataChanged fires when only metadata changed (mtime/ETag)
    // without a content change (size is stable). Consumers should update
    // cached metadata without invalidating content chunks.
    FileEventMetadataChanged FileEventType = "metadata_changed"
)
```

### Logique d'émission par backend

#### WebDAV (`plugins/webdav/webdav.go` — Watch goroutine)

```
Pour chaque entrée dans currentMap :
  old, exists := snapshot[p]
  if !exists:
      → émettre FileEventCreated
         ModTime = fi.ModTime, PreviousModTime = zero
  else if fi.ETag != "" && fi.ETag != old.ETag && fi.Size == old.Size:
      → émettre FileEventMetadataChanged
         ModTime = fi.ModTime, PreviousModTime = old.ModTime
         MetadataOnly = true
  else if fi.ModTime != old.ModTime || fi.Size != old.Size || fi.Version != old.Version:
      → émettre FileEventModified
         ModTime = fi.ModTime, PreviousModTime = old.ModTime
         MetadataOnly = false
```

#### MooseFS (`plugins/moosefs/moosefs.go` — Watch goroutine)

```
Pour chaque entrée dans currentMap :
  old, exists := snapshot[p]
  if !exists:
      → émettre FileEventCreated
         ModTime = fi.ModTime, PreviousModTime = zero
  else if fi.ModTime != old.ModTime && fi.Size == old.Size:
      → émettre FileEventMetadataChanged
         ModTime = fi.ModTime, PreviousModTime = old.ModTime
         MetadataOnly = true
  else if fi.ModTime != old.ModTime || fi.Size != old.Size:
      → émettre FileEventModified
         ModTime = fi.ModTime, PreviousModTime = old.ModTime
         MetadataOnly = false
```

## 4. Proto gRPC — extensions (#130 #131)

Les plugins dynamiques transmettent `FileInfo` et `FileEvent` via gRPC. La couche proto doit être étendue.

### `FileInfoProto` — ajout field 20

```protobuf
message FileInfoProto {
  string name           = 1;
  string path           = 2;
  int64  size           = 3;
  bool   is_dir         = 4;
  int64  mod_time_unix  = 5;
  string etag           = 6;
  bool   is_placeholder = 7;
  bool   is_cached      = 8;
  reserved 9 to 19;
  string version        = 20; // NEW — opaque version token
}
```

### `FileEventProto` — ajout fields 6, 7, 8

```protobuf
message FileEventProto {
  string event_type             = 1;
  string path                   = 2;
  string old_path               = 3;
  int64  timestamp_unix         = 4;
  string source                 = 5;
  int64  mod_time_unix          = 6; // NEW — file's new modtime
  int64  previous_mod_time_unix = 7; // NEW — file's previous modtime
  bool   metadata_only          = 8; // NEW — true = metadata-only change
}
```

**Compatibilité** : les nouveaux champs ont tous des zero-values valides.  
Un plugin binaire v2.0 (sans ces champs) enverrait des zéros → le client gRPC recevrait `ModTime=zero`, `MetadataOnly=false` → comportement identique à l'ancien.

## 6. Critères d'acceptation

- [ ] `FileInfo.Version` populé dans `Stat()` et `List()` pour WebDAV et MooseFS
- [ ] WebDAV `Version == ETag` (même valeur, sans les guillemets)
- [ ] MooseFS `Version == strconv.FormatUint(uint64(attr.CTime), 10)`
- [ ] Watch émet `PreviousModTime` non-zero lors de `FileEventModified` et `FileEventMetadataChanged`
- [ ] Watch émet `ModTime` non-zero (= modtime courant) pour tous les types non-deleted
- [ ] `FileEventMetadataChanged` émis quand ETag change + size stable (WebDAV) ou mtime change + size stable (MooseFS)
- [ ] `MetadataOnly = true` si et seulement si `FileEventMetadataChanged`
- [ ] gRPC : conversion aller-retour fidèle pour tous les nouveaux champs
- [ ] `go test ./plugins/...` passe à 100%
- [ ] Aucun plugin tiers existant ne compile en erreur (zero-value compatible)
