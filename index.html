Pourquoi GhostDrive ?

Le problème

Les solutions actuelles vous enferment dans un écosystème fermé

🔒

Vendor lock-in

Google Drive, OneDrive, iDrive — vos données sont captives de leurs serveurs. Changer de provider signifie tout migrer manuellement, avec des formats propriétaires et des API fermées.

💸

Quota limité et coûteux

Quelques gigaoctets gratuits, puis des abonnements mensuels pour du stockage que vous possédez déjà sur votre NAS ou votre serveur maison.

🔢

Backends privés inaccessibles

Vous avez un NAS, un MooseFS, un serveur WebDAV, un Ceph ? Windows ne sait pas les monter nativement avec l'expérience Files On-Demand. Rclone est puissant mais sans interface, sans placeholders, sans sync automatique.

Notre réponse

La solution GhostDrive

Même expérience qu'OneDrive, vers vos propres backends

Dépendance aux clouds propriétaires → Architecture plugin ouverte — WebDAV, MooseFS, S3, CephFS et plus
Coûts d'abonnement récurrents → Open source, auto-hébergé — vous contrôlez votre infrastructure
Pas de Files On-Demand sur backends privés → Cloud Filter API native Windows — placeholders réels, téléchargement à la demande
Outils complexes sans interface → Interface Windows native avec icône systray, notifications et paramètres visuels
Ce que fait GhostDrive

Fonctionnalités

Tout ce qu'OneDrive offre, vers votre propre stockage

📄

Files On-Demand

Les fichiers apparaissent dans l'Explorateur sans être téléchargés. Accédez-y à la demande via Cloud Filter API — exactement comme OneDrive.

⇋

Sync bidirectionnelle

Les modifications locales se propagent vers le backend, et vice versa. Résolution de conflits intégrée.

💾

Cache local

Activez le cache pour accéder à vos fichiers hors ligne. Synchronisation automatique à la reconnexion.

⚛

Architecture plugin

Chaque backend de stockage est un plugin Go compilé. Ajoutez WebDAV, MooseFS, S3, ou implémentez le vôtre via l'interface StorageBackend.

📦

Distribution simple

Un binaire Windows, pas d'installation complexe. Prérequis : WinFsp pour les placeholders système.

Installation depuis les sources

Aucun binaire publié pour l'instant — le projet est en développement actif

Prérequis

Go 1.21+ Wails v2 Node.js 18+ (frontend) WinFsp (Windows — placeholders)
1
git clone https://github.com/CCoupel/GhostDrive.git
cd GhostDrive
2
# Installer Wails v2
go install github.com/wailsapp/wails/v2/cmd/wails@latest

# Dépendances Go
go mod download

# Dépendances frontend
cd frontend && npm install && cd ..
3
wails dev
ℹ Les binaires précompilés seront disponibles à la release V1 via GitHub Releases.   Suivre les releases sur GitHub
